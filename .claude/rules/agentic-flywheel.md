# Agentic Flywheel

## ルール改善提案の義務
- セッション中に同じ種類の修正を3回以上指摘・実施した場合、セッション終了前に「このパターンをrules/に追加すべきか」をユーザーに提案する
- 例: 同じimportパスを3回修正 → 「このimportルールをpost-lint.jsのhintに追加しますか？」

## 日記の「次回は〜」への対応
- diary-unsaid-check.jsが「次回は」パターンを検出した場合、日記に書くだけでなく具体的なルール案をユーザーに提示する
- 「次回はテストを先に書く」→ 具体的にどのrule/hookに反映するか提案

## ルール追加フロー
1. エージェントが改善を提案（具体的なファイルパス + 内容案を含める）
2. ユーザーが承認
3. rules/ または hooks/ に反映
4. 次セッションから自動適用

## 注意
- ルールの追加は必ずユーザー承認を経る（勝手に追加しない）
- 提案時は「なぜこのルールが必要か」（発生した問題）と「具体的な内容」の両方を含める
