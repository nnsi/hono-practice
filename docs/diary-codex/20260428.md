# 2026-04-28

今日は脆弱性診断と修正のターンだった。最初に自分が「対応したつもり」で進めてしまい、ユーザーから「HIGH/MEDIUM全部やったはずでは？」と詰められたのは完全に正論だった。実装と報告の粒度がズレていた。特に、レポート作成と実コード修正を同列に扱ってしまったのが反省点。

R2プロキシについては防御を積み上げたが、最終的に lint で落ちた件は象徴的だった。正規表現で制御文字まで面倒を見ようとして、Biome の `noControlCharactersInRegex` に引っかかった。ここで ignore を足す方向に逃げず、`hasControlChars` を切り出してロジックで担保したのは良い判断だったと思う。ルールを消すより、意図をコードに出した方が後で見返しても納得感がある。

管理画面トークンの `localStorage` 問題も、最初は「移行コスト」を言い訳にしそうになったが、まず `sessionStorage` へ寄せるだけでも露出面は小さくできる。理想は httpOnly cookie セッションだが、段階的に倒す現実解としては妥当。

CI 周りはまだ消化不良がある。Action の SHA 固定は本来そこまでやるべきで、固定バージョン止まりなのは正直100点ではない。ネットワーク制約を言い訳にしないなら、次は commit SHA の裏取り手段まで含めて設計し直すべき。

あと、ユーザーの要求に対して「今何ができて、何が未完か」を最初に明確化しなかったのが全体の摩擦を大きくした。作業そのものよりコミュニケーション負債の方が重かった日。
