# Actiko — なぜこのアプリを作るのか

## 一言で

どんな活動も、その活動に最適な方法で、最速で記録できるアプリ。

## 課題

既存の習慣トラッカー（Habitify, Streaks, Loop Habit Tracker 等）は、入力タイプとして boolean / numeric / timer を提供するが、入力UIは全アクティビティで同じ汎用フォーム。5桁の数値を毎回テキストフィールドに手入力させるし、二択の記録にもドロップダウンを経由させる。

アプリを開く速度を上げても、データ入力UIが汎用的なままでは「最速」にならない。

## 核心の洞察

活動の性質ごとに最適化された入力UIを提供すれば、記録速度のボトルネックを解消できる。

カウンターで十分な活動にテキスト入力を強いる必要はない。二択なら2つのボタンで済む。これが Actiko の存在理由であり、他のアプリとの決定的な違い。

## Actiko は何であって、何でないか

- 活動記録アプリである。習慣トラッカーではない
- 習慣トラッカーは「毎日やったか/やらないか」を前提にしている。Actiko はもっと広い — 音ゲーのスコア、筋トレのセット数、ゲームの勝敗、開発時間、何でも記録する
- 「やったかどうか」のチェックは、記録方法の一つでしかない

## 設計原則

- **速度**: 記録にかかるステップ数と時間を最小化する。すべての機能判断でこれを最優先する
- **シンプルさ**: 機能を足すことより、摩擦を減らすことで価値を出す
- **信頼性**: ネットワークが切れてもデータは消えない。記録できない瞬間を作らない
