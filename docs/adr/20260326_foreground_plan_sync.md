# フォアグラウンド復帰時のサブスクリプション同期

## ステータス

決定

## コンテキスト

モバイルアプリのサブスクリプション状態（`auth_state.plan`）がローカルSQLiteに保存されており、以下のタイミングでのみ更新されていた:

1. ログイン/新規登録時（`apiGetMe` → DB書き込み）
2. アプリ起動時のトークンリフレッシュ（`serverRefreshAndSync`）
3. RevenueCat経由の購入/復元時

Webで課金した後にiOSアプリをバックグラウンドから復帰させても、ローカルのplanが `free` のままでPro機能が使えない問題があった。

## 決定事項

`AppState` の `active` イベントでフォアグラウンド復帰を検知し、`/user/me` を叩いてplanをローカルDBに同期する。planが `premium` の場合はSiriショートカット用APIキーの自動プロビジョニングも行う。

**スロットリングは入れない。** 理由:

- `/user/me` は軽量（DB 1クエリ）
- planが変わるのは課金時のみのレアイベント
- ユーザー数が少ないフェーズでは負荷が問題にならない
- 実装のシンプルさを優先

負荷が問題になった場合は、最後の同期からN分以内ならスキップするスロットリングを追加する。

## 結果

- フォアグラウンド復帰のたびに `/user/me` が1回発生する
- Web課金後にアプリを開くだけでPro機能が即座に使えるようになった
- Siriショートカット用APIキーがPro確認後に自動でKeychainに保存されるようになった

## 備考

- RevenueCatは未統合のため、現時点ではiOSの「購入を復元」は常に失敗する（想定通り）
- Siri phrasesに `String` パラメータは埋め込めない（Apple制約: `AppEntity`/`AppEnum` のみ）。音声入力は2ステップ対話で対応
