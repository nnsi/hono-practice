# 新規登録画面からユーザー名入力欄を削除

## ステータス

決定

## コンテキスト

新規登録フォームに「ユーザー名（任意）」の入力欄があったが、登録後にユーザー名を表示・利用する画面が存在しなかった。バックエンドAPIでは `name` フィールドはもともとoptionalであり、送信しなくても登録処理に影響はない。

## 決定事項

Web・Mobile両方の新規登録画面からユーザー名入力欄を削除した。

- フロントエンド側の `register` 関数シグネチャから `name` 引数を除去
- APIリクエストボディから `name` フィールドの送信を停止
- バックエンドAPIスキーマは変更なし（`name` はoptionalのまま維持）

## 結果

- 登録フォームの入力項目が1つ減り、登録フローがシンプルになった
- バックエンドは非破壊変更のため、既存のAPI互換性に影響なし
- 将来ユーザー名が必要になった場合は、プロフィール編集画面等で別途対応する
