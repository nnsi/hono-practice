# Actiko プロダクトビジョン

> 最終更新: 2026-03

---

## Actiko とは何か

**どんな活動も、その活動に最適な方法で、最速で記録できるアプリ。**

習慣トラッカーではない。活動記録アプリ。
習慣トラッカーは「毎日やったか/やらないか」を前提にしている。Actiko はもっと広い — 音ゲーのボタン入力回数、筋トレのセット数、ゲームの勝敗、開発時間、何でも記録する。

---

## なぜ Actiko を作るのか

既存の習慣トラッカー（Habitify, Streaks, Loop Habit Tracker 等）は、入力タイプとして boolean / numeric / timer を提供するが、**入力UIは全アクティビティで同じ汎用フォーム**。5桁の数値を毎回テキストフィールドに手入力させるし、二択の記録にもドロップダウンを経由させる。

**データ入力の摩擦が、記録体験のボトルネックになっている。**

アプリを開く速度を上げても、データ入力UIが汎用的なままでは「最速」にならない。アクティビティの性質ごとに最適化された入力UIを提供するアプリは、2026年3月時点で存在しない。

```

│  │  AI: 量の変化の分析・自然言語入力  │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │  可視化: Debt ヒートマップ / 統計   │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │  ゴール & Debt: 目標設定・進捗管理  │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │  ★ 記録モードシステム ★            │  │
│  │  カウンター / バイナリ / テンキー   │  │
│  │  タイマー / チェック / 音声 / ...   │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │  基盤: オフラインファースト + 同期  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

下から順に:

1. **基盤** — オフラインファースト + 自動同期。全てを支える信頼の土台
2. **記録モードシステム** — Actiko の核。活動の性質に合わせた最適な入力UI
3. **ゴール & Debt** — 記録に意味と動機を与える
4. **可視化** — 記録データを「見たくなる形」にする
5. **AI** — 量の変化を分析し、活動間の関連性を伝える。記録をさらに速くする

一番下と真ん中が Actiko のアイデンティティ。上3層は価値を増幅する装置。

## 記録モードシステム

アクティビティ作成時に「どうやって記録するか」を選ぶ。選んだモードに応じて、そのアクティビティ専用の入力UIが表示される。

### カウンターモード

大きな +/- ボタンで数値を増減。刻み幅はカスタマイズ可能（+1/+10/+100/+1000）。

```
┌─────────────────────┐
│                     │
│       1,247         │
│                     │
│   ┌───┐  ┌─────┐   │
│   │ −1│  │ +1  │   │
│   └───┘  └─────┘   │
│   ┌───┐  ┌─────┐   │
│   │−10│  │ +10 │   │
│   └───┘  └─────┘   │
│                     │
│     [ 記録する ]     │
└─────────────────────┘
```

用途: 筋トレ回数、音ゲーのボタン入力回数など数値を積み上げる活動。

### バイナリモード

ActivityKind の選択を2つの大きなボタンに昇格。1タップで即記録。

```
┌─────────────────────┐
│                     │
│  ┌────┐   ┌────┐   │
│  │ 勝 │   │ 負 │   │
│  │ ち │   │ け │   │
│  └────┘   └────┘   │
│                     │
│    今日: 3勝 2敗     │
│                     │
└─────────────────────┘
```

用途: ゲームの勝敗など二択の記録。画面を開いた時点で当日の集計が見える。

### テンキーモード

アプリ内テンキーで大きな数値を直接入力。OS キーボードを経由しない。

```
┌─────────────────────┐
│          38,247     │
│                     │
│  ┌───┬───┬───┐      │
│  │ 7 │ 8 │ 9 │      │
│  ├───┼───┼───┤      │
│  │ 4 │ 5 │ 6 │      │
│  ├───┼───┼───┤      │
│  │ 1 │ 2 │ 3 │      │
│  ├───┼───┼───┤      │
│  │ C │ 0 │ ⏎ │      │
│  └───┴───┴───┘      │
└─────────────────────┘
```

用途: セッション終了後にスコアや累計値を入力する場面。

### タイマーモード

開始→バックグラウンド計測→停止→記録。

```
┌─────────────────────┐
│                     │
│      01:23:45       │
│                     │
│   [ 開始 / 停止 ]    │
│                     │
│     [ 記録する ]     │
└─────────────────────┘
```

用途: 開発時間、勉強時間、読書時間など継続的な活動。

### チェックモード

「やったか/やらないか」の1タップ記録。

```
┌─────────────────────┐
│                     │
│      ┌─────┐        │
│      │  ✓  │        │
│      └─────┘        │
│                     │
│  今月: 18/30日       │
│                     │
└─────────────────────┘
```

用途: 瞑想、日記、薬を飲んだなど。従来の習慣トラッカーが対象とする領域。Actiko ではこれは記録モードの一つでしかない。

### 音声モード

音声入力 → AI がアクティビティと数値を自動判定 → 確認して記録。

```
┌─────────────────────┐
│                     │
│      🎤 聞いてます   │
│                     │
│  「腕立て20回」      │
│                     │
│  → 腕立て / 20回     │
│    [ 記録する ]      │
└─────────────────────┘
```

用途: 手が塞がっている場面（運動中、料理中）。AI による解析はオンライン時に実行。オフライン時はテキストをキューに保存し、次回接続時に処理する。よく使うパターンはローカルにキャッシュし、LLM を呼ばずに即時マッピングする。

## 記録の入り口

アプリを開かなくても記録できる手段を複数のレイヤーで提供する。

### レイヤー0: 自動記録（将来）

カレンダー連携、ヘルスケア連携など。記録行為自体をなくす。

### レイヤー1: ウィジェット / 通知アクション（アプリ外から1タップ）

ホーム画面・ロック画面のウィジェットに記録モードを配置する。

- カウンターウィジェット: +1 ボタンを常時表示
- バイナリウィジェット: 勝ち/負けボタンを常時表示
- チェックウィジェット: ✓ ボタンを常時表示
- 通知アクション: 常駐通知のアクションボタンから記録

### レイヤー2: アプリ内ショートカット（アプリ起動後すぐ）

- 起動直後に直前に使ったアクティビティの記録モードを表示
- よく使うアクティビティのピン留め

### レイヤー3: 通常の記録フロー

アクティビティ一覧から選んで記録。現在の動線。

## ゴール & Debt システム

記録に意味と動機を与えるレイヤー。

- **日次目標量**: アクティビティごとに1日の目標量を設定
- **Debt（負債）**: 未達分が負債として蓄積。翌日以降の超過で返済できる
- **ストリーク**: 連続達成日数の追跡
- **バランス計算**: 目標に対する貯金/負債の推移

## 可視化

記録したデータを「毎日見たくなる形」にする。

### Debt ヒートマップ

GitHub コントリビューショングラフと同じ形式。カレンダー上に達成/未達を色分け表示。

```
     3月
月 ■■□■■■□■■■□■■■
火 ■■■■□■■■■□■■■■
水 ■□■■■■□■■■■□■■
木 ■■■■■□■■■■■□■■
金 □■■■□■■■□■■■□■
土 ■■■■■■■■■■■■■■
日 ■■■□■■■■□■■■■■

■ = 達成  □ = 負債
```

- 空白を埋めたくなる心理を活用
- Free: 直近1ヶ月 / Pro: 全期間

### Debt 返済グラフ

負債残高の推移を折れ線グラフ化。返済トレンドが一目でわかる。

### 完済予測線

現在のペースで返済を続けた場合の完済予定日。

### ストリーク表示

連続達成日数をアニメーション付きで表示。途切れると鎮火。

### 統計チャート

日次/月次/四半期/年次の集計。活動量の推移。

## AI

記録された「量」を分析し、活動間の関連性をユーザーに伝えるレイヤー。AI は量の変化を伝えるだけで、質の評価や行動の良し悪しは判断しない。

### 自然言語入力

テキストや音声からアクティビティと数値を自動判定する。記録モードシステムの一部として機能する（音声モード）。

### 週次レビュー

1週間のログデータを要約し、活動間の量の変化を分析する。

例:
- 「先週は筋トレの回数が前週比 +15%。一方で読書の時間は -20%」
- 「筋トレをした日は音ゲーの入力回数が平均 8% 多い傾向があります」

### 朝ブリーフィング

今日のゴールと昨日の負債をプッシュ通知で伝える。

例:
- 「今日の目標: 筋トレ3セット、読書30分。昨日の負債: 読書15分」

### パターン検出

長期的なログデータから、活動間の量の相関を検出する。

例:
- 「水曜に筋トレを休むと、木曜の音ゲー入力回数が平均 5% 減少する傾向」

## 基盤: オフラインファースト + 自動同期

全ての体験を支える信頼の土台。

- **IndexedDB (Dexie.js) がプライマリDB**: サーバーはレプリカ。ネットワーク切断中も全機能が動作
- **バックグラウンド同期**: 30秒間隔 + ネットワーク復帰時の自動同期
- **依存順序付き同期**: データの整合性を保証
- **マルチデバイス**: 同一アカウントで複数端末から記録・閲覧
- **データは消えない**: オフライン中の記録も復帰時に確実に同期される

この基盤があるから、ウィジェット・Watch・通知アクションなどアプリ外からの記録が信頼できる。
