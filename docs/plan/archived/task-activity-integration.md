# Task × Activity 連携アイデア

> 作成: 2026-03-13
>
> 前提:
> - Activity は「やったことを記録する」コア機能。ここは変えない
> - Task は「今日何するんだっけ」を管理する計画レイヤー
> - 現状 Task と Activity は完全に独立。リンクなし
> - Actiko のユーザーは「やった後にアプリを開いて記録する」フローが主。「やる前にアプリを開いて何をするか決める」のは Task の役割

---

## 1. Task ↔ Activity リンク（基盤）

タスク作成時に、任意でアクティビティを紐付けられるようにする。

```
タスク: 「読書 30分」
  └→ リンク: 読書 (Activity)
```

全てのアイデアの土台。`task` テーブルに `activity_id uuid REFERENCES activity(id)` (nullable) を追加。リンクなしの純粋な TODO も引き続き作れる。

---

## 2. タスク完了 → 記録ショートカット

Activity にリンクされたタスクを完了した時、そのまま記録モードを開く導線を出す。

```
✅ 読書 30分 を完了

  ┌─────────────────────┐
  │ 読書を記録しますか？   │
  │                     │
  │ [ 記録する ] [ 後で ] │
  └─────────────────────┘
```

**なぜ良いか**: 「タスク完了」と「活動記録」が1フローになる。記録忘れが減る。「やる前にアプリを開いてタスクを確認する」ユーザーが、自然に記録まで到達する導線になる。

---

## 3. ゴールからの自動タスク提案

`day_targets` で今日が目標日になっている Activity のタスクを自動提案する。

```
📋 今日のタスク
─────────────
  □ MTGの準備          ← 手動で作ったタスク
  □ メール返信          ← 手動で作ったタスク
  ─ ─ ─ ─ ─ ─ ─ ─ ─
  💡 目標から
  □ 読書 30分          ← ゴールから自動生成（負債 90分）
  □ 筋トレ 3セット     ← ゴールから自動生成
```

手動登録不要。タスクを見に来た時に Activity のゴールが自然に視界に入る。提案であって強制ではない。

---

## 4. Debt コンテキスト表示

Activity リンク付きタスクに、Debt 情報を添える。

```
□ 読書 30分
  負債 90分 │ 今日やれば → 60分に減少
```

タスクを見る時点で「これをやる意味」が分かる。記録後の Debt フィードバック（`debt-continuation-ideas.md` Tier 1）と対になる。

---

## 5. 振り返り: 計画 vs 実績

日次/週次で「タスクで計画したこと」と「実際に記録したこと」を並べる。

```
📊 今日の振り返り

  計画(Task)        実績(Activity)
  ─────────        ──────────
  読書 30分 ✅      読書 45分（+15分貯金）
  筋トレ 3set ✅    筋トレ 3set
  メール返信 ✅     （Activity なし）
  MTG準備 ❌       音ゲー 2000（計画外）
```

「やろうとしたこと」と「実際にやったこと」のギャップが見える。Stats / AI レイヤーでさらに発展可能。

---

## 実装順序

| 順序 | アイデア | コスト | 依存 |
|-----|---------|-------|------|
| 1 | Task ↔ Activity リンク | 小（カラム追加 + UI） | なし |
| 2 | タスク完了 → 記録ショートカット | 小（UI導線追加） | 1 |
| 3 | ゴールからの自動タスク提案 | 中（day_targets 参照 + 提案ロジック） | 1 + Debt 基本実装 |
| 4 | Debt コンテキスト表示 | 小（表示追加） | 1 + Debt 基本実装 |
| 5 | 振り返り: 計画 vs 実績 | 中〜大（集計 + UI） | 1 |

1 → 2 で最小コストで「タスク完了 → そのまま記録」の体験が作れる。3 / 4 は Debt 実装後に自然に足せる。
