## 未対応 Medium

### 1. `since`パラメータ未検証 (4/5同期エンドポイント)

- **対象**: `/users/v2/activity-logs`, `/users/v2/goals`, `/users/v2/tasks`, `/users/v2/notes`
- **問題**: `since` が `new Date()` に直接渡され、不正値で500。本番ではスタックトレースは返さない (`honoWithErrorHandling.ts` で `NODE_ENV !== "production"` ガード済み) ため情報漏洩はないが、不要な500がモニタリングノイズになる
- **参考**: `goalFreezePeriodSyncRoute.ts` は `z.string().datetime().optional().safeParse()` で検証済み
- **修正方針**: 同パターンを残り4エンドポイントに適用

### 2. リフレッシュトークンがレスポンスbodyに露出

- **対象**: `/auth/login`, `/auth/token` のJSON応答
- **問題**: httpOnly Cookieと同時にbodyにも `refreshToken` を返しており、WebでXSS時に窃取可能
- **修正方針**: User-Agentまたはカスタムヘッダーでプラットフォーム判定し、Webクライアントにはbody返却を省略

### 3. レートリミットがKV未設定時に無効化

- **対象**: `/auth/login`, `/auth/google`, `/auth/apple`, `/user` (登録), `/contact`
- **問題**: `RATE_LIMIT_KV` がない環境でブルートフォース制限なし。`X-Forwarded-For` スプーフ可能
- **修正方針**: (a) インメモリフォールバック or KV未設定時にログ警告、(b) 本番は `CF-Connecting-IP` を使用

### 4. Base64アイコンアップロードにサイズ制限なし

- **対象**: `ActivityIconUploadRequestSchema` の `base64: z.string().min(1)`
- **問題**: `.max()` なしで巨大ペイロードによるメモリ枯渇 (DoS)
- **修正方針**: `.max(7_000_000)` (約5MB base64) を追加
