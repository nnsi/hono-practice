## 0. 前提: プロダクトビジョンとの関係

`product-vision.md` で定義した Actiko の構造:

```
  5. AI              ─┐
  4. 可視化           │ 価値増幅レイヤー（Pro）
  3. ゴール & Debt    ─┘
  ─────────────────────
  2. 記録モード       ─┐ アイデンティティ（Free）
  1. オフライン基盤   ─┘
```

**マネタイズの境界線は、アイデンティティと価値増幅レイヤーの間に引く。**

- レイヤー 1-2（基盤 + 記録モード）: Actiko が Actiko である理由。制限しない
- レイヤー 3-5（ゴール拡張 + 可視化 + AI）: 記録データの価値を増幅する装置。ここで課金する

---

## 1. Free / Pro の境界

### 1.1 原則

- **記録体験は聖域**: どの記録モードも、何件のアクティビティも、何件のログも制限しない
- **増幅レイヤーで課金**: 記録した後の「理解・分析・示唆」に対して対価を得る
- **味見させてから売る**: Pro 機能は完全にブロックせず、制限付きまたはプレビューで体験させる

### 1.3 音声モードの位置づけ

音声モードは記録モードの一つ（レイヤー2）だが、AI（レイヤー5）に依存する。

- Free: 1日3回まで。記録モードとしての体験を保証しつつ、Pro への自然な導線になる
- Pro: 無制限。LLM コストは Pro 月額で吸収

3回/日の根拠: 「毎日使って良さを体感できるが、ヘビーに使うには足りない」ライン。

### 1.4 Activity Kinds の位置づけ

Activity Kinds は記録モード（レイヤー2）ではなくゴール管理（レイヤー3）に属する。記録の入力UIを変えるのではなく、データの分類粒度を上げる機能。バイナリモード（勝ち/負け）は Activity Kinds なしでも記録モードとして動作する。

## 2. 価格設定

### 2.1 価格

| | 月額 | 年額 | Lifetime (初期限定) |
| --- | --- | --- | --- |
| JPY | ¥680 | ¥5,800 | ¥9,800 |
| USD | $4.99 | $39.99 | $69.99 |

### 2.2 根拠

- **¥680/月**: 競合（Habitify ¥600, Todoist ¥588）のやや上。AI のランニングコストを含む差額として説明可能。¥700 は超えない
- **年額 29% 割引**: 月額換算 ¥483。「月500円以下」の訴求力。チャーンを構造的に下げる
- **Lifetime ¥9,800**: 約14ヶ月分。早期支援者向け。一定数で販売終了。LTV 毀損を防ぐため常設にしない

### 2.3 ストア手数料

| チャネル | 手数料 | 月額の実収入 |
| --- | --- | --- |
| Web (Stripe) | 3.6% | ¥655 |
| iOS/Android (1年目) | 30% | ¥476 |
| iOS/Android (2年目〜) | 15% | ¥578 |

Web 課金を優先導線にする。ストア価格は据え置き（手数料は集客コスト）。年額を強く推す。

## 3. 実装順序: 売るものを先に作る

### Phase 0: 記録モードシステム

**Pro を売る前に、Free の体験を完成させる。**

Actiko のアイデンティティである記録モードが未実装のまま課金を始めても、ユーザーが定着しない。定着しなければ課金もしない。

```
0-a. 記録モードの実装
     - カウンターモード（刻み幅カスタマイズ）
     - バイナリモード（1タップ即記録）
     - テンキーモード（アプリ内テンキー）
     - タイマーモード（バックグラウンド計測）
     - チェックモード（やった/やらない）
     - アクティビティ作成時のモード選択UI

0-b. 記録の入り口拡張
     - ウィジェット（iOS/Android）
     - 通知アクション
     - アプリ内ショートカット（ピン留め / 前回のモード即表示）
```

ここが完成した時点で、Actiko は「活動に最適化された入力UIを持つ唯一のアプリ」になる。Free ユーザーの定着基盤ができる。

### Phase 1: Pro の中身を作る

**課金に値する体験を完成させる。**

```
1-a. AI 機能
     - 音声モード（Web Speech API / Expo Speech + LLM）
     - オフラインキュー + ローカルキャッシュ（頻出パターン即時解決）
     - AI 週次レビュー（量の変化 + 活動間相関）
     - AI 朝ブリーフィング（今日の目標 + 昨日の負債）
     - パターン検出（長期データの相関分析）

1-b. Debt 可視化強化
     - Debt ヒートマップ（GitHub 風カレンダー）
     - Debt 返済グラフ（折れ線）
     - 完済予測線
     - ストリーク表示強化

1-c. 詳細分析
     - 週次 / 四半期 / 年次チャート
```

### Phase 1 完了基準（課金開始ゲート）

以下の KPI を満たすまで課金を開始しない。

| KPI | 目標 | 意味 |
| --- | --- | --- |
| 音声モード利用率 | DAU の 20%+ | AI 記録が使われているか |
| AI 週次レビュー開封率 | 配信数の 50%+ | レビューに価値を感じているか |
| Debt ヒートマップ閲覧頻度 | 週3回+/ユーザー | 可視化が習慣に組み込まれているか |

未達なら課金を延期し、機能の UX を見直す。

### Phase 2: 課金基盤

```
2-a. Stripe 統合（Checkout + Customer Portal + Webhook）
2-b. usePlanGate フック + Free 制限 UI（ブラー + アップグレード導線）
2-c. 14日間 Pro Trial（カード不要、登録時自動開始）
2-d. プラン状態の Dexie キャッシュ（オフライン対応）
2-e. RevenueCat（iOS/Android ストア課金）
```

### Phase 3: ストア + グロース

```
3-a. iOS App Store / Google Play ローンチ
3-b. グロース施策の順次投入
```

## 4. Free → Pro 転換設計

### 4.1 転換ファネル

```
[登録]
  → 記録モードで最速記録を体感（定着）
    → データが蓄積する
      → 可視化・AIの価値に気づく（ヒートマップ、音声モード上限、週次レビュー）
        → Pro Trial
          → 課金
```

**Phase 0（記録モード）がファネルの入口**。ここでの定着率が全てを決める。

### 4.2 転換トリガー

| タイミング | 条件 | メッセージ | 表示方法 |
| --- | --- | --- | --- |
| 音声モード上限 | Free で3回/日の上限到達 | 「音声で記録し放題。Pro を試してみませんか？」 | 入力欄の下にインライン |
| 2つ目のゴール | Free でゴール追加を試みた | 「複数のゴールを同時に追跡」 | ダイアログ内 |
| ログ 100 件到達 | 累計 ≧ 100 | 「AI にパターン分析させてみませんか？」 | ホーム画面バナー |
| 初回 Debt 発生 | ゴール未達 | 「Debt 返済グラフで推移を確認する」 | ゴール詳細画面 |
| ヒートマップ表示 | 1ヶ月分の端に到達 | 「全期間のヒートマップを見る」 | ヒートマップ下部にブラー |
| 月次統計閲覧 | 統計ページを開いた | 「週次・年次で長期トレンドを見る」 | チャート下部にブラープレビュー |

### 4.3 Trial

- 登録時に14日間 Pro Trial を自動開始（カード不要）
- Trial 中は全 Pro 機能利用可能（AI 含む）
- 残り3日 / 1日 / 終了時にバナー。終了時は「失うもの」を具体的に提示
- 終了後: Free にダウングレード。**データは絶対に消さない。機能のみロック**

### 4.4 ダウングレード体験

Pro → Free に戻ったユーザーが「やっぱり Pro がほしい」と思えるかが転換率を決める。

- Pro で作った Activity Kinds → **閲覧可、編集不可**
- 2つ目以降のゴール → **閲覧可、新規作成不可**
- Debt 返済グラフ → **直近1週間のみ表示、以降ブラー**
- AI 週次レビュー → **最後のレビューを表示し続ける**。「次のレビューは Pro で」
- ヒートマップ → **直近1ヶ月のみ。古い部分はブラー**

「ブロック」ではなく「ここから先は Pro」。データの存在は見えるが、フルアクセスにはProが必要。

## 5. 課金ガード設計

### 5.1 サーバーサイド

```
premiumMiddleware (既存) でガード:
  - AI 関連エンドポイント（週次レビュー、パターン検出、朝ブリーフィング）
  - POST /users/api-keys
  - 音声モード: Free でも呼べるが、日次上限をサーバーでカウント

plan: 'free' | 'premium'
status: 'trial' | 'active' | 'cancelled' | 'expired'
Grace Period: 決済失敗後 7日間は premium 維持
```

### 5.2 クライアントサイド

```
usePlanGate フック:
  - plan 状態を Dexie にキャッシュ（オフライン対応）
  - 同期時にサーバーの subscription 状態で更新
  - ネットワーク切断中は最後のキャッシュを信頼

ゲーティング表示:
  - 「❌ 使えません」ではなく「Pro ならここが見えます」+ ブラー/プレビュー
  - 記録モード（レイヤー2）には一切ゲートを入れない
```

### 5.3 AI コスト管理

```
モデル選択:
  - 音声モード（自然言語解析）: Haiku（低コスト・高速）
  - 週次レビュー: Sonnet（品質重視）
  - パターン検出: Haiku + 統計前処理（LLM は解釈のみ）
  - 朝ブリーフィング: テンプレート生成（LLM 不要。Debt計算 + ゴール取得のみ）

コスト抑制:
  - 週次レビュー: 週1回生成 → キャッシュ（何度でも閲覧可）
  - パターン検出: 月1回バッチ生成 → キャッシュ
  - 音声モード: 頻出パターンをローカルキャッシュ（LLM 呼び出し回避）
  - 閾値: ユーザーあたり月 ¥100 超で管理者アラート
```

## 7. リスクと対策

| リスク | 深刻度 | 対策 |
| --- | --- | --- |
| **記録モードが定着しない**（Phase 0 失敗） | 致命的 | 自分自身で dogfooding。記録モードの UX を徹底的に磨く。ここが失敗したら課金以前の問題 |
| **AI 機能の利用率が低い** | 高 | Phase 1 完了基準で検証。未達なら課金延期。音声モードの UX（特にオフラインキュー）を重点改善 |
| **Free で十分と感じるユーザーが多い** | 中 | 音声モード3回/日の体験枠 + ダウングレード体験の設計で対応 |
| **LLM API コスト高騰** | 中 | OpenRouter 経由でモデル切り替え可能。キャッシュ + ローカルパターンマッチで呼び出し回数抑制 |
| **Apple/Google 手数料** | 中 | Web 課金を優先導線。手数料は集客コストとして許容 |
| **競合が入力UI最適化を真似る** | 低 | 記録モード + Debt + AI の組み合わせは模倣コストが高い。先行者優位を活かす |

## 8. まとめ

### 原則

1. **アイデンティティは無料**: 記録モードシステム + オフライン基盤は一切制限しない
2. **増幅レイヤーで課金**: 記録データの「理解・分析・示唆」に対して対価を得る
3. **定着が先、課金は後**: Phase 0（記録モード）→ Phase 1（Pro 機能）→ Phase 2（課金基盤）の順
4. **味見させてから売る**: 音声モード3回/日、ヒートマップ1ヶ月分、ブラープレビュー
5. **データで判断**: Phase 1 完了基準を満たさなければ課金を延期する
